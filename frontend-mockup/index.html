Clients

Le contexte customer détient l'identité, le segment et le statut KYC. Un client est l'agrégat racine qui possède un ou plusieurs comptes du contexte account.

Maquette d'anticipation — contexte à venir

Ces écrans préfigurent le futur module customer (CLOSED, dépend de shared). Le lien client → comptes se ferait via un identifiant customerId exposé par le contexte account, sans couplage de modèle entre les deux BCs (relation par référence, éventuellement orchestrée par événements CustomerOnboarded / AccountOpened).
